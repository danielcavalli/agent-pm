## Organization

The ADRs are grouped by theme:

**Data & Storage (001-005, 024):** How pm stores, structures, identifies, and indexes project data. ADR-024 documents the shift from global to local-first storage (superseding ADR-002).

**Architecture & Interfaces (006-009, 030):** How the four interfaces (CLI, MCP, slash commands, TUI) are layered and how they interact with the data layer. ADR-030 explains why epics are the sole organizational construct.

**Implementation (010-014, 025, 031):** Technology choices for the implementation -- language, validation, CLI framework, TUI framework, error handling, file locking, and I/O strategy.

**Agent Lifecycle (026-029):** How agents communicate, report, and how the system synthesizes and garbage-collects their artifacts.

**Integration & Deployment (015-018):** How pm is installed, configured across multiple AI clients, and how agent behavior is shaped.

**Evolution (002, 019-020, 022):** Transitional and superseded decisions for migrating from earlier architectures and documenting deprecated components.

**Future Architecture (021, 023):** Accepted and proposed architectural direction for upcoming releases, including agent-aware TUI and swarm self-improvement.
