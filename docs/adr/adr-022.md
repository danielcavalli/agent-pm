# ADR 022: Dropping the pm-reminder Compaction Plugin

## Status

Accepted

## Context

The original PRD (Section 7.4) specified an optional `pm-reminder` compaction plugin (`install/opencode/plugins/pm-reminder.ts`) that hooked into OpenCode's `experimental.session.compacting` lifecycle event. When an agent's context window was being compacted, the plugin would inject a reminder to file any discovered but unfiled bugs, tech debt, or improvements using `pm_story_add` or `pm_epic_add`. This was tracked as story PM-E018-S002 ("Create optional pm-reminder compaction plugin") within the AGENTS.md Rules and Plugin Hook epic (PM-E018).

The plugin depended on the `@opencode-ai/plugin` SDK, which provided access to OpenCode's internal session lifecycle hooks. When the project migrated from the custom OpenCode plugin system to MCP as the universal tool interface (ADR 007, ADR 020), the `@opencode-ai/plugin` dependency was removed entirely. MCP does not expose client-specific session lifecycle events like context compaction -- it defines a tool invocation protocol, not a hook system. There is no MCP equivalent of `experimental.session.compacting`.

Meanwhile, the AGENTS.md rules (ADR 017) and the session-reminder paragraph installed into `~/.claude/CLAUDE.md` by `install.sh` provide the same functional intent through a different mechanism: they instruct agents to file discoveries before context is lost. Rather than a programmatic hook that fires during compaction, the filing guidance is embedded in the agent's instruction context from the start of every session. This approach works across all MCP-compatible clients, not just OpenCode.

## Decision

The pm-reminder compaction plugin was dropped without replacement. PM-E018-S002 was marked as `cancelled` with the rationale that the underlying plugin infrastructure (OpenCode's `@opencode-ai/plugin` SDK and `experimental.session.compacting` hook) no longer exists.

The autonomous filing behavior that the plugin was meant to encourage is instead covered by:

1. **AGENTS.md rules** (installed by `install.sh`): Define when to file (bugs, tech debt, missing features, missing tests, performance concerns), when not to file (related to current task, trivial, uncertain), and how to file (check status, choose epic vs story, use clear titles, continue current task).

2. **Session-reminder paragraph** (appended to `~/.claude/CLAUDE.md` for Claude Code): Explicitly reminds the agent to file discoveries before context is compacted.

No code was written for the plugin -- it was cancelled before implementation began. The related cleanup epic PM-E027-S004 ("Migrate pm-reminder plugin to MCP-compatible approach") chose option (b): remove references and document that AGENTS.md rules serve as the primary autonomous filing guidance.

## Consequences

The AGENTS.md-based approach has one advantage over a compaction hook: it operates continuously rather than only at compaction time. Agents are instructed to file discoveries as they encounter them during normal work, not just when the system signals an imminent context loss. This reduces the risk of an agent accumulating discoveries and then running out of context before the compaction event fires.

The tradeoff is that there is no programmatic enforcement. A compaction hook was a guaranteed trigger point -- the system itself would prompt the agent. With AGENTS.md rules, the agent must self-regulate based on its instruction context. In practice, this has been sufficient: agents working under the pm filing rules consistently file discoveries during normal task execution without needing a system-level prompt.

If a future MCP specification introduces client lifecycle events (e.g., a standardized `context.compacting` notification), revisiting a programmatic reminder mechanism may be warranted. Until then, the instruction-based approach through AGENTS.md rules remains the supported method.
