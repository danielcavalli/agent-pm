# Execution Report Contract

## Purpose

This contract defines the format and content requirements for execution reports generated by agents. An execution report captures the outcome of executing a story (task), including decisions made, assumptions relied upon, tradeoffs considered, and observations that were out of scope. The consolidation agent uses these reports to detect conflicts between parallel agents, promote important decisions into Architecture Decision Records (ADRs), and build shared context for future work.

## Schema

Reports are validated against the `AgentExecutionReportSchema` defined in `src/schemas/agent-report.schema.ts`.

## Required Fields

### Identification

- **`task_id`** (string, required): The story code being executed. Must match pattern `PROJECT-ENNN-SNNN` (e.g., `PM-E030-S001`).
- **`agent_id`** (string, required): Identifier for the agent that completed the task. Use a short descriptive string (e.g., `claude-agent-1`). Max 100 characters.
- **`timestamp`** (string, required): ISO 8601 datetime when the report was created (e.g., `2026-03-11T10:45:00Z`).

### Status

- **`status`** (enum, required): Report completion status. Must be one of:
  - `complete` -- Task fully completed
  - `partial` -- Task only partially completed

## Optional Fields

All array fields default to empty arrays and accept a maximum of 10 items each. Text fields within items are capped at 500 characters.

### decisions (array of DecisionItem)

Choices made and their rationale during implementation.

- **`type`** (enum, required): `"episodic"` (historical narrative for consolidation context) or `"semantic"` (ADR candidate, eligible for promotion).
- **`text`** (string, required, max 500 chars): Description of the decision.

### assumptions (array of AssumptionItem)

Priors the agent relied on that were not validated.

- **`type`** (enum, required): `"episodic"` or `"semantic"`.
- **`text`** (string, required, max 500 chars): Description of the assumption.

### tradeoffs (array of TradeoffItem)

Alternatives considered and rejected.

- **`alternative`** (string, required, max 500 chars): The alternative that was considered.
- **`reason`** (string, required, max 500 chars): Why it was rejected.

### out_of_scope (array of OutOfScopeItem)

Observations that surfaced but were not acted on.

- **`observation`** (string, required, max 500 chars): What was observed.
- **`note`** (string, optional, max 500 chars): Additional context.

### potential_conflicts (array of PotentialConflictItem)

Self-flagged assumptions the agent knows are uncertain or likely to conflict with other agents' work. Implements the Reflexion verbal self-reflection pattern, converting episodic uncertainty into a prior signal for the consolidation agent.

- **`assumption`** (string, required, max 500 chars): The uncertain assumption.
- **`confidence`** (enum, required): `"low"`, `"medium"`, or `"high"`.
- **`note`** (string, optional, max 500 chars): Additional context.

### consolidated (boolean)

Set to `true` by the consolidation agent after this report is ingested. Used for filtering and garbage collection eligibility. Defaults to `false`.

## Output Format

Reports are serialized as YAML sidecar files in `.pm/reports/` with the naming convention `<story-id>-report.yaml` (e.g., `PM-E030-S001-report.yaml`).

Example report matching the `AgentExecutionReportSchema`:

```yaml
task_id: PM-E030-S001
agent_id: claude-agent-1
timestamp: "2026-03-11T10:45:00Z"
status: complete
decisions:
  - type: episodic
    text: Placed schema file in src/schemas/ alongside existing project and story schemas
assumptions:
  - type: episodic
    text: Assumed max 10 items per array field is sufficient based on typical story complexity
tradeoffs:
  - alternative: Use JSON Schema directly
    reason: Zod integrates better with existing codebase and provides compile-time types
out_of_scope:
  - observation: Report CLI command is not yet registered in cli.ts
    note: Tracked in PM-E045-S002
potential_conflicts:
  - assumption: Report file naming uses story code as prefix
    confidence: high
    note: Other agents may expect a different naming convention
consolidated: false
```

## Creating Reports

### CLI

```
pm report create \
  --task-id <STORY_CODE> \
  --agent-id <YOUR_ID> \
  --status <complete|partial> \
  --decisions "type:text" \
  --assumptions "type:text" \
  --tradeoffs "alternative|reason" \
  --out-of-scope "observation|note" \
  --potential-conflicts "assumption|confidence|note"
```

### MCP Tool

Use the `pm_report_create` tool with `workdir` set to the repo root.

## Validation Rules

- `task_id` must match pattern `PROJECT-ENNN-SNNN` (e.g., `PM-E030-S001`)
- `agent_id` must be 1-100 characters
- `timestamp` must be a valid ISO 8601 datetime string
- `status` must be `"complete"` or `"partial"`
- All array fields accept a maximum of 10 items
- All text/string fields within items are capped at 500 characters
- `type` fields in decisions and assumptions must be `"episodic"` or `"semantic"`
- `confidence` in potential_conflicts must be `"low"`, `"medium"`, or `"high"`
