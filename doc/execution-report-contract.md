# Execution Report Contract

## Purpose

This contract defines the format and content requirements for execution reports generated by agents. An execution report captures the outcome of executing a story (task), including success/failure status, timing information, and any error details.

## Required Information

Every execution report MUST capture the following fields:

### Identification

- **`story_code`**: The PM story code being executed (format: `PROJECT-ENN-SNNN`, e.g., `PM-E030-S001`)
- **`id`**: Unique execution ID (format: `XNNNNNN`, e.g., `X000001`)

### Status Tracking

- **`status`**: Current execution state. Must be one of:
  - `pending` — Task queued but not started
  - `running` — Task in progress
  - `completed` — Task finished (regardless of outcome)
  - `failed` — Task encountered a fatal error
  - `cancelled` — Task was cancelled before completion

- **`result`**: Outcome of the execution. Must be one of:
  - `success` — Task completed successfully
  - `failure` — Task failed to complete
  - `cancelled` — Task was cancelled
  - `timeout` — Task exceeded time limit

### Timestamps

- **`started_at`**: ISO 8601 datetime when execution began (e.g., `2026-03-11T10:30:00Z`)
- **`completed_at`**: ISO 8601 datetime when execution finished (optional if still running)

### Context

- **`executor`**: Identifier of the agent or system that executed the task (optional)
- **`error_message`**: Error details if the execution failed (optional, default: empty string)
- **`output`**: Console output, logs, or other execution artifacts (optional, default: empty string)

## Output Format

Reports MUST be serialized as JSON matching the `ExecutionReportSchema`:

```json
{
  "id": "X000001",
  "story_code": "PM-E030-S001",
  "status": "completed",
  "result": "success",
  "started_at": "2026-03-11T10:30:00Z",
  "completed_at": "2026-03-11T10:45:00Z",
  "executor": "agent-001",
  "error_message": "",
  "output": "..."
}
```

## Agent Responsibilities

When an agent completes story execution, it MUST:

1. Generate a unique execution ID following the `XNNNNNN` pattern
2. Set `started_at` to the actual start time of execution
3. Set `completed_at` to the actual finish time of execution
4. Capture any error messages if the result is `failure`
5. Include relevant console output in the `output` field
6. Use `pm report create` CLI command to persist the report

## Validation Rules

- `story_code` must match pattern `PROJECT-ENN-SNNN`
- Datetime fields must be valid ISO 8601 strings
- `completed_at` must be greater than or equal to `started_at`
- `result` is required when `status` is `completed` or `failed`
